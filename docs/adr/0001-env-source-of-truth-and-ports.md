# ADR 0001: Environment source of truth and local ports

## Context
Project startup and docs referenced mixed local backend ports (`3000` and `3001`) and mixed env ownership between root `.env` and `backend/.env`.

`backend/src/config/env.ts` loads dotenv from root (`../../../.env`), but `run.bat` previously did not ensure `backend/.env` existed.

## Decision
- Standardize local backend port to `3000`.
- Set root `.env` as the source of truth for environment values.
- Keep `backend/.env` as compatibility copy generated by `run.bat` when missing.
- Align `.env.example` callback/API URLs with port `3000`.

## Consequences
- Startup behavior is predictable across backend, frontend proxy, and scripts.
- New developers can copy `.env.example` and run without immediate port mismatches.
- Existing tooling expecting `backend/.env` keeps working due to compatibility copy.
